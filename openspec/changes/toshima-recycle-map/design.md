## Context

現在の `src/routes/+page.svelte` は単一のMapLibreマップに静的マーカー1個を表示するのみ。豊島区の資源回収拠点は乾電池・廃食油・小型家電・蛍光灯・インクカートリッジの5品目に分かれ、各施設は複数品目を扱う場合がある。将来は東京23区全体に拡張し、データ量増加に対応できる設計が必要。

## Goals / Non-Goals

**Goals:**
- GeoJSONによる静的データ管理で豊島区の拠点を地図表示する
- 品目フィルタUIで表示するマーカーを絞り込む
- 将来の東京全体拡張・外部DB移行に耐えるデータスキーマを定義する
- 住所からの緯度経度変換ワークフローを確立する

**Non-Goals:**
- リアルタイムデータ更新・管理画面
- ルート案内・ナビゲーション
- 他区のデータ収集（Phase 1では豊島区のみ）

## Decisions

### D1: データ形式 — GeoJSON静的ファイル（将来PMTilesへ移行可能）

**理由**: MapLibreはGeoJSONをネイティブサポートし、追加依存なしで動作する。豊島区の拠点数は数十件程度でファイルサイズは問題なし。

**将来拡張**:
- 東京全区（数千件規模）→ PMTiles（ベクタータイル形式、CDNで配信可）
- リアルタイム更新が必要 → Supabase + PostGIS（`ST_AsGeoJSON`でGeoJSONとして返却）

**採用しなかった代替案**:
- SQLite/Turso: エッジ対応だがSvelteKitのSSGと相性が悪い
- PocketBase: 小規模向きだが自前ホスティングが必要

### D2: GeoJSONスキーマ — `ward`フィールドで区を識別

```json
{
  "type": "Feature",
  "geometry": { "type": "Point", "coordinates": [経度, 緯度] },
  "properties": {
    "id": "toshima-001",
    "ward": "toshima",
    "name": "施設名",
    "address": "豊島区○○1-2-3",
    "categories": ["battery", "small-appliance"],
    "hours": "月〜金 8:30〜17:15",
    "notes": "補足事項"
  }
}
```

`ward`フィールドにより東京全区のデータを単一GeoJSONまたは区別ファイルに統合できる。`categories`は配列で複数品目を保持。

**カテゴリ識別子**:
| キー | 品目 |
|------|------|
| `battery` | 乾電池 |
| `fluorescent` | 蛍光灯 |
| `cooking-oil` | 廃食油 |
| `ink-cartridge` | インクカートリッジ |
| `small-appliance` | 小型家電 |

### D3: ジオコーディング — 国土地理院APIをビルド時に使用（決定済み）

**エンドポイント**: `https://msearch.gsi.go.jp/address-search/AddressSearch?q=<住所>`

**動作確認済み（2026-04-11）**:
- 「豊島区池袋3-29-10」（「東京都」なし・省略形）→ `[139.7061, 35.736084]` ✓
- 「東京都豊島区巣鴨3-15-20」→ `[139.733917, 35.735512]` ✓
- 番地レベルの精度、APIキー不要、無料・無制限

住所は区公式サイトから取得した省略形のまま投げられる。ビルド時スクリプトで座標を付与しGeoJSONに保存し、実行時のAPIコールは不要。Nominatimは採用しない。

### D4: フィルタ状態管理 — Svelte 5 `$state`

グローバルストアは使わず、`+page.svelte`の`$state`でフィルタ選択を管理し`$derived`でフィルタ済みのGeoJSONを生成する。

### D5: ヘッダーとタイポグラフィ — 見出しは Dela Gothic One を採用（決定）

見出し（タイトル）にはモダン太ゴシック調の日本語フォント「Dela Gothic One」を使用する。本文は既存のSans系（例: Noto Sans JP/システムフォント）を用い、見出し用途に限定する。

ガイドライン:
- 文言: 「東京リサイクルマップ：豊島区版」（将来は `{区名}` 可変）
- 配置: 画面最上部の固定ヘッダー（sticky, 高Z-index）
- サイズ: モバイル18–20px / デスクトップ22–24px、行高1.2–1.3
- 色/コントラスト: 背景白に#111〜#222、必要に応じて境界線/影
- フォント配信: Google Fontsを第一候補（`font-display: swap`、`preconnect`推奨）。将来は自前ホスト検討可
- アクセシビリティ: タイトルは見出し要素として扱い、読み上げで意味が伝わること

## Risks / Trade-offs

- **座標精度リスク**: 番地レベルの精度を確認済み。ただし存在しない番地や新住所は失敗する可能性がある → スクリプト実行時にエラー行をログ出力し手動補正
- **データ鮮度**: 区の公式サイト更新時に手動でGeoJSONを更新する必要がある → 将来的にスクレイピングCIで自動化
- **PMTiles移行コスト**: ファイル分割・ビルドパイプライン変更が必要 → `ward`フィールドとカテゴリスキーマを最初から設計しておくことで移行コストを最小化
