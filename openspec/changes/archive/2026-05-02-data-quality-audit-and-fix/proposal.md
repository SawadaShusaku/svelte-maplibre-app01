## Why

現在のGeoJSONデータセットに深刻な品質問題が存在する。特に、**47件の施設名・住所に文字化け（`` 混入）**が発生しており、ユーザーが地図上で直接目にする情報が破壊されている。例として `shinagawa-051` の「`螢戰螢読霏⊂山店`」など、施設名が完全に読めない状態になっている。

さらに、品川区（shinagawa.geojson）に「東京都稲城市長峰1丁目1番地」の住所を持つ「総合体育館」という施設が登録されており、市区町村と住所が明らかに不整合である。また、不完全な住所（「北区」「杉並区」など区名のみ）が52件、重複住所が119組存在する。

これらの問題は手動で発見されており、データセットが拡大するにつれて品質管理が困難になる。データの信頼性を確保し、**人間が正しいデータを提供するまで誤データがビルドに流れ込まない仕組み**が必要である。

## What Changes

- **データ品質監査スクリプトの作成**: 全GeoJSONファイルに対して以下を自動検出するNode.jsスクリプトを追加する。ただし、**スクリプトは検出のみ行い、データの自動修正は一切行わない**。
  - 文字化け（``、無効なUTF-8シーケンス）の検出
  - ファイル名と `properties.city` の不整合
  - `id` の重複およびフォーマット違反
  - 必須プロパティの欠損
  - 古いカテゴリID（`battery` 等）の混入
  - 同一座標・同一住所による重複施設の検出（ファジーマッチング対応）
  - 住所の完全性チェック（区名のみ、番地欠落等）
- **修正必要データ一覧表の自動生成**: 検出した問題を、人間がコピーして修正作業に使えるMarkdown表形式で出力する。列：ファイル、ID、施設名、住所、問題の種類、備考。
- **既知の誤データのビルドブロック**: 文字化けや市区町村不整合が検出された場合、`npm run build:db` を停止し、「以下のデータに問題があります。修正データをご提供ください」と明確に報告する。
- **データ検証フローの確立**: `npm run build:db` 実行前に品質チェックを実行する仕組みを追加。問題検出時はビルドを失敗させ、人間が修正したGeoJSONを受け取ってから再実行する。

## Capabilities

### New Capabilities
- `geojson-audit`: GeoJSONデータ品質の自動監査と検証スクリプト（検出のみ、自動修正なし）
- `data-validation-pipeline`: ビルド前データ品質チェックの統合フロー（人間レビュー必須）

### Modified Capabilities
- `recycle-data`: データ品質チェック要件の強化と新規検証シナリオの追加

## Impact

- `src/lib/data/**/*.geojson` — **人間が修正した正しいデータに差し替え**（AIは書き換えない）
- `scripts/` — 新規データ監査スクリプトの追加
- `package.json` — 新規npmスクリプト（例：`npm run audit:data`）の追加
- `npm run build:db` — ビルド前に品質チェックを実行。問題未解決時はビルド失敗
- データ品質問題が未解決の場合、ビルドが失敗する（意図的な設計）
