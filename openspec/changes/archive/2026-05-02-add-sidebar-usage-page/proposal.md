## Why

サイドバーの「使い方」は優先度の高い情報項目として並んでいるが、現状は開ける先がなく、初めて使う人が地図の見方や絞り込み操作を自己解決しづらい。掲載区が増えて操作対象も広がっているため、現在の画面構成に対応した案内ページを用意し、実際の機能とずれない説明を提供する必要がある。

## What Changes

- サイドバーの `使い方` 項目から開ける、アプリ内の利用ガイドページを追加する
- 利用ガイドで、区の選択、カテゴリ絞り込み、地図マーカーの見方、表示設定の使い方を案内する
- 利用ガイドの説明内容が実際の UI と掲載データ範囲に一致するよう、誤解を招かない文言要件を定義する
- サイドバーの情報項目のうち `使い方` を利用可能な導線として接続し、未実装項目との見え方を区別できるようにする

## Capabilities

### New Capabilities
- `sidebar-usage-page`: サイドバーの `使い方` から開ける利用ガイドページの表示内容、導線、説明対象を定義する

### Modified Capabilities
- `sidebar-navigation`: `使い方` 項目を実際に遷移可能な情報導線として扱う要件に更新する

## Impact

- `src/lib/components/SettingsSidebar.svelte` の情報メニュー導線
- `src/routes/` 配下の新規ガイドページとルーティング
- 既存の `openspec/specs/sidebar-navigation/spec.md` と、新規の利用ガイド向け spec
- 将来の `データについて` `更新情報` `プライバシーポリシー` 追加時の情報ページ実装パターン
