## Why

区データが増えたことで、サイドバーは単なる仮置きメニューではなく、設定と案内導線を整理して支える役割が必要になっている。現状はマーカーデザイン設定とプレースホルダー項目が混在しており、何を調整できる場所なのか、どこに情報を探しに行けるのかが分かりにくい。

## What Changes

- サイドバーを「表示設定」と「情報・ヘルプ」の2系統に整理し、役割の異なる項目を明確に分離する
- 既存のマーカーデザイン設定をサイドバーの主設定として見直し、初見でも触り方が理解しやすい表示に整える
- 情報系メニューを優先順位付きで整理し、`使い方`、`データについて`、`更新情報`、`プライバシーポリシー` を中心に導線を定義する
- `掲載区一覧` は追加しない。掲載範囲は地図とフィルタの表示から把握できる前提を維持する
- 将来的に情報系メニューを別URLのページへ遷移できるよう、リンク型メニューとして拡張しやすい構造にする

## Capabilities

### New Capabilities
- なし

### Modified Capabilities
- `sidebar-navigation`: サイドバー内の情報設計、設定項目の見せ方、情報系メニューの導線要件を更新する

## Impact

- `src/lib/components/SettingsSidebar.svelte` のレイアウト、セクション構成、メニュー表現
- サイドバーの開閉元である `src/lib/components/AppHeader.svelte` と連携する文言・導線
- 将来の情報ページ追加に向けた、サイドバーメニュー項目の構造整理
- UI 要件として、設定系と案内系の役割分離、優先度順の情報導線、追加予定ページへの拡張余地を明文化
